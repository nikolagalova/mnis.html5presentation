Class model

Návrh diagramu tried bol vytvorený v nástroji Enterprise Architect. Applikácia je rozdelená do viacerých vrstiev od UI až po perzistentnú vrstvu. Časť UI zahŕňa triedy obsluhujúce jednotlivé stránky - ManagedBeans, ktoré sú anotované ako @ViewScoped. Samostatnou triedou je SessionManager, ktorého inštancia existuje počas celej session (@SessionScoped). Beany ďalej pracujú s DAO triedami (data access object), ktoré narábajú so samotnými entitami.
